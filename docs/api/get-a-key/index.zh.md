# 申请 API 密钥

本章节的教程至少需要一家服务商的 API 密钥才能运行。我们覆盖的三家服务商都遵循 **OpenAI 的接口协议**，所以一份 `openai` Python 客户端代码就能覆盖它们全部 —— 你只需要换 `base_url` 和 `model`。

## 服务商速览

| 服务商 | 需要安装的 SDK                       | 适合                           | 备注                                        |
|--------|-------------------------------------|--------------------------------|--------------------------------------------|
| OpenAI   | `openai`                          | 生态最广，示例最多                | 使用前需要先充值                              |
| DeepSeek | `openai`（走兼容端点）          | 低成本，推理能力强              | 复用 `openai` 客户端，改 `base_url` 即可     |
| Qwen     | `openai`（兼容） 或 `dashscope` | 有免费额度，也有开源权重         | 通过 DashScope 提供（阿里云账号）         |

!!! tip "先挑一家开始"
    如果拿不定主意，从 **OpenAI**（网上示例最多） 或 **DeepSeek**（最便宜） 入手。以后再补其他的也可以 —— [统一客户端](../unified-client.md) 这个模式用同一份代码就能覆盖三家。

## 妥善保管密钥

不论是哪家，规则都一样：

- 把密钥放在环境变量或 git 忽略的 `.env` 文件里 —— 绝不要写死在代码里。
- 如果密钥曾经被粘到共享文档、已提交的 notebook 单元格或截图里，立刻轮换这把密钥。
- 条件允许时，按项目分别生成密钥，这样需要吊销时代价低。

有关本站使用的 `.env` 做法，参见 入门 章节里的 [API 密钥](../../getting-started/api-keys.md)。
